# v0.1.2 — Add Market Support to Document Forensics

## Summary

The forensics service was hardcoded for Vietnamese (VN) documents only. This change adds a `market` parameter that adjusts OCR language selection, field classification regex rules, and Gemini prompt language per country.

## Supported Markets

| Code | Country    | OCR Languages | Prompt Language      |
|------|------------|---------------|----------------------|
| VN   | Vietnam    | `vi,en`       | Vietnamese           |
| TH   | Thailand   | `th,en`       | Thai                 |
| HK   | Hong Kong  | `ch_tra,en`   | Traditional Chinese  |
| ID   | Indonesia  | `id,en`       | Indonesian           |

Default: `VN` (backward compatible).

## Files Changed

### New
- `agents/document-forensics/extraction/market-config.ts` — `MarketCode`, `MarketConfig`, per-market field rules, `resolveMarket()`, `getMarketConfig()`

### Modified
- `agents/document-forensics/extraction/easyocr-extractor.ts` — replaced hardcoded `FIELD_RULES`, `getLangs()`, name regexes with `MarketConfig`
- `agents/document-forensics/extraction/gemini-extractor.ts` — parameterized `buildSystemPrompt()` with `promptLanguage`
- `agents/document-forensics/forensics.ts` — threaded `market` through `advancedDocumentForensics`, `runHybridForensics`, `extractDocumentFields`, `batchDocumentForensics`
- `agents/document-forensics/handler.ts` — added `market?: string` to `AnalyzeRequest`, `BatchRequest`, `ExtractFieldsRequest`

## Backward Compatibility

- `market` is optional on all request interfaces
- `resolveMarket(undefined)` returns `'VN'` — existing callers get identical behavior
- VN field rules are the exact same regexes that were previously hardcoded
- `process.env.EASYOCR_LANG` override still takes precedence over market config

## Call Chain

```
POST /forensics/analyze { image_path, market: "TH" }
  → handler.ts: handleAnalyze(req)
    → forensics.ts: advancedDocumentForensics(..., rawMarket="TH")
      → resolveMarket("TH") → MarketCode = "TH"
      → runHybridForensics(..., market="TH")
        → new EasyOCRExtractor(undefined, "TH")
          → langs = "th,en" (from MarketConfig)
          → classifyText() uses TH fieldRules
        → OR new GeminiExtractor(undefined, "TH")
          → prompt = "expert in Thai and English..."
```
